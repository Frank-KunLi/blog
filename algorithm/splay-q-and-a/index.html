求前驱后继时为什么要取等号？

Q: 以Splay::lower()的代码为例，在u更新时，我们使用的是e[u].val <= e[u].val ? 0 : 1而非val < e[u].val ? 0 : 1，为什么？

A: 根据平衡树的性质，小于当前节点的数在左边，而e[u].val的前驱显然不能是e[u].val本身，而是在其左子树中，所以要写等于号。

为什么你除了 find() 以外的操作也要 splay 到根？

Q: 我看的洛谷题解就不是这样的！你这样不会拖慢常数吗？

A: 你让题解过一下洛谷第12个点？

A: 这样子做能使经常被访问的点提高到根，尽量避免 Splay 被卡，然而还是可能会被良心出题人针对。。。
